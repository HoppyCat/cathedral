## On the Five Questions

### 1. Session Bootstrap

The `CLAUDE.md` approach works, but there's a nuance: **CLAUDE.md is read as instructions, not executed automatically.** Claude will follow the instruction to call `GET /api/context` at session start, but it relies on Claude reading and acting on it — which it reliably does, but it's not a true system hook.

The better primitive is **Claude Code's `startup` behavior via CLAUDE.md** combined with a **hooks entry in `.claude/settings.json`**. Specifically:

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "",
      "hooks": [{"type": "command", "command": "curl -s -H 'X-Galaxie-Secret: $GALAXIE_SECRET' https://your-api/api/context > /tmp/galaxie-context.json"}]
    }]
  }
}
```

Actually the cleanest approach: put the bootstrap call in CLAUDE.md as an explicit first-turn instruction *and* store the secret in environment so it's available. The `/api/context` shape proposed is good. **Leaner alternative:** split it into tiers — always load `mood + workingMemory`, load `soul` only when the task is identity-sensitive.

### 2. Mid-Session Writes and Session Close

**Yes, Claude Code can make authenticated HTTP requests** — via the Bash tool using `curl` with headers. As long as `GALAXIE_SECRET` is in the environment (set via `.env` or `settings.local.json`), calls like this work fine mid-session:

```bash

  -H "X-Galaxie-Secret: $GALAXIE_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"content": "...", "target": "working"}'
```

For **session close** — Claude Code has a `Stop` hook that fires when Claude finishes a response. It's not a true "window closes" event, but it's close enough for writing a session note. However, the CLAUDE.md instruction approach ("before ending, POST /api/session/close") is actually *more reliable* because Claude will do it intentionally rather than via a hook that might fire mid-task.
